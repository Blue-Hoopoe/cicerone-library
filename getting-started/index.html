About

The word "cicerone" means more or less a "pilot". This library provides flexible tools, which allow to simply guide users on your websites by displaying personalized dialogs. Personally, I use it to present new features of my applications, but I truly believe, that it has a little bit more uses.

Let's asume, that your website has a new feature in the form of social share button. User can miss this important element of your page, unless you directly point it with your finger. Cicerone library creates half-transparent curtain on your site and step-by-step shows selected elements with your commentary in the form of previously created template.

Initialize presentation

After including css and js files, create new instance of Cicerone object with array of objects that defines every step of your presentation.

This one contains only one: it will point the #first-element-to-show element and will use template saved in #dialog-template" <script> element.

After creating an object, the presentation is invoked by a start() method.

<script>
    var myPresentation = new Cicerone([
        {
            selector: "#first-element-to-show",
            template: "#dialog-template"
        }
    ]);
    myPresentation.start();
</script>

Templates

<script type="text/template">
    <h1 cicerone-next> Hello {{ library }}</h1>
</script>

script>
    var myPresentation = new Cicerone([
        {
            selector: "#first-element-to-show",
            template: "#dialog-template",
            variables: {
                "library": "Cicerone"
            }
        }
    ]);
</script>

Templates must be stored in <script type="text/template"> tag. It should contains ordinary html content, that is latter added to dialog body.

What's important, Cicerone library handles simple variable inserting. The syntax can be known among others from Twig, but it is limited to only simple objects like strings or numerics.

There is also possibility to add navigation listeners to templates, which will be used to manipulate the presentation (e.g. "next" button). To add listener, include one of the tree attributes to the chosen element:

  • cicerone-next - shows next dialog,
  • cicerone-back - shows previous dialog,
  • cicerone-end - ends the presentation,

Ciceron object

Cicerone instance contains three public methods:

  • start(from = 1) - initialize the whole presentation. The from parameter defines from which step the presentation should be open (by default it is the first one).
  • makeStep(step) - makes next or previuos steps in the presentation depending on the step parameter.
  • end() - finishes the presentation and remove all elements with cicerone class.

There is also one public property:

  • steps - contains all steps in the presentation. Can be used to remove or add new one.

Step objects

As mentioned, the Cicerone constructor takes array of steps objects. Step objects must contains properties:

  • selector - css selector to the presented element,
  • template - css selector to the <script type="text/template"> element, which holds the inner-dialog template.

Additionaly, it can also have:

  • css - style string that will be add to presented element during it's step.
  • placement - additional class that will be added to dialog. By deafult, library handles two values: left (default) and right,
  • variables - object of simple objects variables that will be inserted to the tamplate. It is handled like a associative array with 'variable-name'=>'value' structure. 
<script>
    var myPresentation = new Cicerone([
        {
            selector: "#first-element-to-show",
            template: "#dialog-template"
        },
        {
            selector: "#second-element-to-show",
            template: "#dialog-template"",
            css: "background: yellow; color: red;"
        },
        {
            selector: "#third-element-to-show",
            template: "#dialog-template"
            variables: {
                "header": "Third element",
                "content": "3 dialog in the presentation."
            },
            placement: "right"
        }
    ]);
</script>
